# Neon Survivors Prototype — Design Brief

## 1. 프로젝트 비전
- **장르 & 콘셉트**: 네온 터미널 테마의 탑다운 생존 슈터. Vampire Survivors 류의 자동 전투 감성을 ASCII HUD와 결합.
- **플랫폼**: 데스크톱 브라우저 (Canvas 2D 기반). 모바일 대응, 사운드는 미구현.
- **목표 경험**: 짧은 세션 동안 점점 조이는 적 밀도 속에서 자원 수집과 자동 성장의 시너지를 체험.

## 2. 핵심 사용자 여정
1. `index.html`을 열면 즉시 시뮬레이션 부팅.
2. 플레이어는 `@` 포인터를 WASD/방향키로만 조작.
3. 무기는 가장 가까운 적을 자동 조준·발사, 처치 시 전리품 드롭.
4. 드롭 아이템으로 XP/버프/회복을 얻고, 자동 레벨업으로 파워 스파이크.
5. 시간이 지날수록 새로운 적 계층과 스폰 속도가 가속 → 체력 0시 세션 종료.

## 3. 게임 시스템 사양
### 3.1 플레이어
- **기본 수치**: 체력 100, 이동속도 280px/s, 기본 탄환 피해 20, 발사 간격 750ms.
- **레벨업 보상**: 최대 체력 +6, 체력 12 회복, 탄환 피해 +4, 발사 간격 40ms 단축(최소 320ms 부근까지).
- **버프 지속시간**: 오버클럭 6.5초(발사 간격 55% 적용), 페이즈 쉴드 5초(받는 피해 40%로 감소). 중복 획득 시 남은 시간 중 최댓값 유지.

### 3.2 전투 & 조작
- **조작**: 키보드 이동만 지원. 자동 사격 방향 계산, 자동 탄환 발사.
- **탄환 처리**: 최근접 적을 탐지해 460px/s 직선 이동. 적과 충돌 시 피해 적용 후 탄환 삭제.
- **피해 시스템**: 적 접촉 피해는 유형별 고정. 쉴드 중에는 60% 피해 경감.

### 3.3 적 웨이브 디자인
| 적 유형 | 등장 시점 | 특징 | 전리품 XP |
| --- | --- | --- | --- |
| Glitch Bit | 0초 | 기본 근접 추격, 중형 체력. | 12 |
| Daemon Wisp | 20초 | 지그재그 이동, 높은 속도, 낮은 체력. | 14 |
| Kernel Bruiser | 45초 | 느리지만 높은 체력/피해. | 28 |
| Torrent Brood | 60초 | 소형 다중 스폰(3기), 중간 속도. | 10 × 3 |

- **스폰 로직**: 가장자리 랜덤 생성, 경과 시간에 따라 유효 적 풀 확장.
- **난이도 곡선**: 스폰 주기(기본 1500ms)가 생존 시간에 비례해 360ms까지 감소. 개별 적 속도 또한 시간 스칼라를 더해 상승.

### 3.4 전리품 테이블
| 드롭 | 확률 | 효과 |
| --- | --- | --- |
| 데이터 조각(XP) | 적 기본 드롭 | 적 XP 값을 여러 조각으로 분배하여 총량 지급. |
| 에너지 셀 | 14% | 체력 +14 (최대 체력까지 회복). |
| 오버클럭 코어 | 10% | 6.5초 동안 발사 간격 ×0.55. |
| 페이즈 쉴드 | 8% | 5초 동안 받는 피해 ×0.4. |

- 아이템은 초기 속도로 흩어졌다가 감속하며, 플레이어 140px 이내에서 자력 흡입.

### 3.5 HUD & UX
- **표시 요소**: 생존 시간, 처치 수, 체력 %, 레벨, XP 게이지, 활성 버프 목록.
- **피드백**: 상태 로그가 버프/레벨업/회복 이벤트마다 메시지 출력, 일정 시간 후 "Awaiting input..."으로 복귀.
- **게임 오버**: "Process Terminated" 배너 노출 및 Enter/클릭 재시작.

## 4. 기술 구성
- **렌더링**: HTML `<canvas>` + 12px 격자 ASCII 렌더링(문자/색상 2중 버퍼 후 커밋).
- **데이터 구조**: 적/탄환/아이템/효과 상태를 단일 `state` 객체에서 관리.
- **루프**: `requestAnimationFrame` 기반 델타 타임 보정. 물리 업데이트 → 충돌 판정 → HUD 동기화 → ASCII 커밋.
- **배포**: 정적 한 파일(HTML/CSS/JS 통합). GitHub Pages에 즉시 호스팅 가능.

## 5. 현재 제한사항
- 오디오, 업그레이드 선택 UI, 모바일 터치 입력, 세이브/메타 진행은 미구현.
- 난이도 커브는 시간 기반 선형 감소만 적용, 웨이브 이벤트/보스 없음.
- 접근성: 자막/색약 모드 미지원, 키 리바인딩 미구현.

## 6. 개발 로드맵 제안
1. **세션 다양화**: 웨이브 이벤트, 보스, 시나리오 미션 추가.
2. **메타 시스템**: 라운드 종료 보상, 영구 업그레이드, 무기 빌드 분기.
3. **피드백 강화**: 사운드, 화면 효과, 버프 타이머 표시.
4. **UX 확장**: 모바일 가상 스틱, 키 바인딩, 접근성 옵션.
5. **코드 구조화**: 모듈 분할, 타입 정의, 테스트 유틸 도입.

## 7. 실행 방법
1. 저장소 클론 후 `index.html` 단일 파일을 브라우저로 열기.
2. 페이지 로드시 즉시 게임 시작 → WASD/방향키로 이동.
3. 세션 종료 후 Enter 또는 캔버스 클릭으로 재시작.

> **빌드 없음**: 패키지 설치나 빌드 스텝 없이 GitHub Pages에 바로 배포 가능합니다.
